# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: Tailwind v4 @theme vs :root conflict — CSS vars in `:root` (e.g. `--color-siag-navy`) do NOT auto-generate Tailwind utility classes (`bg-siag-navy`). Only `@theme` vars do. BUT adding the same variable name to both `@theme` and `:root` causes a PostCSS build failure. Use inline `style={{}}` or `bg-[#hex]` for one-off cases instead of @theme for variables already in :root.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Tailwind v4 @theme duplicate | Defining `--color-siag-navy` in both `:root` and `@theme` → PostCSS build failure | blocking | Use `style={{ backgroundColor: '#003B5E' }}` or `bg-[#003B5E]` for colors already in `:root`; only use `@theme` for NEW tokens not in `:root` |
| Prisma 7 url in schema.prisma | `url = env("DATABASE_URL")` in datasource block → validation error in Prisma 7 | advisory | Move all datasource URLs to `prisma.config.ts` using `defineConfig` |
| bg-siag-navy transparent header | Using `bg-siag-navy` as Tailwind class when it's not in `@theme` → transparent header in both modes | advisory | Verified with compiled CSS grep before assuming class works |
