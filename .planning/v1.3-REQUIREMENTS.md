# v1.3 Requirements — Design Modernization & Tech Debt Cleanup

## Goal

Moderne, kohärente Enterprise-Dashboard-Ästhetik nach Design-System; responsive Mobile UX; strenge Tech Debt Governance.

**Priorität:** Design-First (70%) + Tech Debt (30%)  
**Status:** Locked nach Questioning Phase (2026-04-19)

---

## Design-Refresh (70% Fokus)

### D1: Design-System Implementation
- Alle UI-Komponenten konform mit Design-System-Spec
  - Inter Font, #f0f2f5 Background, #CC0033 Swiss Red Accents
  - Floating Red Pill Navbar als Signature Element
  - Box-shadow only (no borders on cards), pill-shaped buttons (border-radius: 9999px)
  - Severity Badges: Critical/High/Medium/Low/Info mit spezifischen Farb-Kombinationen
  - Platform Badges: Windows/macOS/Linux/iOS/Android/Server
  - Status Indicators: green/amber/red dots mit ::before pseudo-elements
  - Toast Notifications: bottom-right, dark charcoal bg, auto-dismiss ~3.5s
  - Slide-in Detail Panel: 480px width, right-aligned, red header

### D2: Mobile Responsiveness
- Floating Pill Navbar responsive: stacking/collapsing auf Tablet/Mobile
- Content Grid-Layouts: 1-col mobile, 2-col tablet, 3-col desktop
- Tables scrollbar-friendly: horizontal scroll oder card view toggle
- Slide-in Detail Panel: 480px → 100vw-safe auf Mobile
- Touch-friendly button sizes: min 44px tap target

### D3: Navigation & Information Architecture
- Horizontale Pill-Navbar als primäre Navigation (kein Sidebar mehr)
- View-Switching via pill buttons (Dashboard, Incidents, Playbooks, Settings)
- Search input mit Expansion on focus
- Badge Counters auf Buttons für Quick Overview
- Sub-tabs innerhalb Views (z.B. Incident Details: Timeline/Activity/Attachments)

### D4: View-spezifische Redesigns
- **Dashboard:** Stat Cards mit 3px Accent Bars, Chart Cards, Incident Overview
- **Incident Details:** Slide-in Panel mit red header, dt/dd rows (label + value), muted labels
- **Incident Wizard:** Step-by-step visual improvements, final report modal (centered, 860px max-width)
- **Settings:** 210px left nav + 1fr content layout, active items with red bg (#fff5f7)
- **Incident List:** Table mit filter pills, sortable columns, pagination

## Tech Debt Cleanup (30% Fokus)

### T1: GitHub Secrets Configuration
- `.env.example` Template mit allen Required Vars (DATABASE_URL, VERCEL_TOKEN, API_KEYS, etc.)
- Local `.env.local` für Developer (in .gitignore)
- GitHub Secrets Setup für CI/CD (DATABASE_URL, VERCEL_TOKEN, API_KEYS)
- Documentation: .env Setup Guide in README

### T2: Branch Protection Rules (main branch)
- Mindestens 1 PR Review erforderlich
- Status Checks erzwungen (Tests müssen grün sein)
- Admin Enforcement (prevent force pushes)
- Dismiss stale PR approvals: enabled

### T3: @swagger Annotations für neue Routes
- App Router routes (`/app/api/**`) mit @swagger TypeDoc comments
- Request/Response Schemas dokumentieren
- Path parameters, query parameters, request body documented
- Success responses (200/201) + error responses (400/404/500) documented
- Legacy API bleibt unverändert (keine retrospektiven Annotations)
- OpenAPI Spec auto-generated und deployable
