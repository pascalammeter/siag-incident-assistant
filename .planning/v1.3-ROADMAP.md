# v1.3 Roadmap — Design Modernization & Tech Debt Cleanup

**Milestone:** v1.3 — Design Modernization & Tech Debt Cleanup  
**Duration:** ~9 phases (4–6 weeks estimated)  
**Priority:** Design-First (70%) + Tech Debt (30%)

---

## Phase Breakdown

### Phase 20: Tech Debt — GitHub Secrets & Branch Protection
**Goal:** Secure environment variable management + enforced code review workflow  
**Scope:**
- `.env.example` Template mit allen Required Vars
- GitHub Secrets Configuration (DATABASE_URL, VERCEL_TOKEN, API_KEYS)
- Branch Protection Rules auf main (1 review required, status checks enforced)
- Documentation: .env Setup Guide in README

**Deliverables:**
- `.env.example` file
- GitHub Secrets configured
- Branch protection rules active
- README.md updated with .env setup

**Dependencies:** None

---

### Phase 21: Tech Debt — Swagger Annotations & OpenAPI Docs
**Goal:** Complete OpenAPI documentation for new App Router routes  
**Scope:**
- @swagger TypeDoc comments auf `/app/api/**` routes
- Request/Response schemas documented
- Path/query parameters documented
- Success/error responses (200/201/400/404/500)
- OpenAPI Spec auto-generated

**Deliverables:**
- Swagger-annotated routes
- OpenAPI Spec (generated)
- Swagger UI endpoint live
- Integration guide updated

**Dependencies:** Phase 20

---

### Phase 22: Design System Setup — CSS Tokens & Variables
**Goal:** Establish design system tokens (colors, typography, shadows, spacing)  
**Scope:**
- CSS Custom Properties (--red, --bg, --surface, --shadow, --radius, etc.)
- Typography scale (display-lg, body-md, etc.)
- Color palette (primary red, grays, semantic colors)
- Box-shadow tokens (--shadow-sm, --shadow, --shadow-lg)
- Spacing scale (4px base)

**Deliverables:**
- `globals.css` with :root CSS variables
- Design token documentation
- Tailwind @theme section updated
- No hardcoded colors in codebase

**Dependencies:** None

---

### Phase 23: Dashboard Redesign
**Goal:** Modernize dashboard layout with stat cards, charts, incident overview  
**Scope:**
- Stat Cards mit 3px Accent Bars (red, green, amber, gray)
- Chart Cards (fixed 180px height, responsive width)
- Incident Overview table (responsive, filterable)
- Floating Red Pill Navbar integration
- Mobile responsiveness (1-col on mobile, 2-col tablet, 3-col desktop)

**Deliverables:**
- Dashboard UI redesigned
- Responsive grid layouts
- Stat cards with accent bars
- Charts integrated
- Mobile-tested

**Dependencies:** Phase 22

---

### Phase 24: Incident Details & Slide-in Panel
**Goal:** Redesign incident detail view with slide-in panel for context  
**Scope:**
- Slide-in Detail Panel: 480px width, right-aligned, red header
- dt/dd rows for metadata (label + value)
- Status indicators (green/amber/red dots)
- Severity badges (Critical/High/Medium/Low/Info)
- Platform badges (Windows/macOS/Linux/iOS/Android/Server)
- Smooth transitions (cubic-bezier)
- Responsive on mobile (480px → 100vw-safe)

**Deliverables:**
- Incident Detail Page redesigned
- Slide-in Panel component
- Badge components (severity, platform, status)
- Mobile-tested

**Dependencies:** Phase 22, 23

---

### Phase 25: Wizard Redesign
**Goal:** Modernize incident wizard with improved step-by-step UX  
**Scope:**
- Step-by-step visual improvements (progress indicators, spacing)
- Final Report Modal (centered, 860px max-width, red header, smooth animations)
- Form styling (pill buttons, error states, validation feedback)
- Toast notifications (bottom-right, auto-dismiss ~3.5s)
- Mobile responsiveness

**Deliverables:**
- Wizard UI redesigned
- Modal component for final report
- Toast notification system
- Form validation styling
- Mobile-tested

**Dependencies:** Phase 22, 24

---

### Phase 26: Settings & Navigation Overhaul
**Goal:** Redesign settings view and floating pill navbar  
**Scope:**
- Floating Red Pill Navbar: #CC0033 bg, white text, pill-shaped buttons, search input
- Settings Sidebar: 210px left nav + 1fr content layout
- Active nav item styling (red bg #fff5f7, red text #CC0033)
- Sub-tabs within views (Incident Details: Timeline/Activity/Attachments)
- View switching via pill buttons (Dashboard, Incidents, Playbooks, Settings)

**Deliverables:**
- Floating Pill Navbar component
- Settings layout (sidebar + content)
- Sub-tabs component
- Navigation routing updated
- Mobile-tested (navbar stacks/collapses)

**Dependencies:** Phase 22, 25

---

### Phase 27: Mobile Responsiveness & Polish
**Goal:** Full mobile UX optimization across all views  
**Scope:**
- Viewport tests: 375px (mobile), 768px (tablet), 1024px (desktop)
- Touch-friendly button sizes (min 44px tap target)
- Table scrolling behavior (horizontal scroll oder card view toggle)
- Floating navbar responsive stacking
- Slide-in Panel responsive (480px → 100vw-safe)
- Scrollbar styling (::-webkit-scrollbar)

**Deliverables:**
- Mobile viewport tests passed
- Touch-friendly interactions
- Responsive table layouts
- Scrollbar styling applied
- All views mobile-tested

**Dependencies:** Phase 23–26

---

### Phase 28: Design System Validation & E2E QA
**Goal:** Complete design system validation, accessibility audit, UAT  
**Scope:**
- WCAG AA contrast testing (WAVE tool)
- Design system spec compliance review (screenshot comparison)
- E2E tests updated for new UI layouts
- Integration tests for new components
- UAT with stakeholders
- Bug fixes & refinements

**Deliverables:**
- WCAG AA compliance verified
- Design system spec compliance checklist
- E2E tests passing
- UAT sign-off
- Release notes prepared

**Dependencies:** Phase 20–27
