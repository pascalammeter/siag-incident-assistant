## Critical Gaps (Blockers)

### GAP-1: PDF Export Not Wired in App Router
- **Affected:** P1.1–P1.6, B5.1–B5.4
- **Root cause:** `src/app/api/export/route.ts` does not exist. Express PDF handler at `src/api/routes/incidents.ts:512` is not mounted in the Next.js App Router.
- **Current state:** `IncidentList.handleExportClick` is a `console.log` stub. `Step6Dokumentation` calls `window.print()` as fallback only.
- **Fix:** Create `src/app/api/incidents/[id]/export/pdf/route.ts` that calls the existing `PDFService` / Puppeteer logic.

### GAP-2: IncidentService Discards Wizard Payload
- **Affected:** B4.1–B4.5, W1.1–W1.5
- **Root cause:** `src/api/services/incident.service.ts createIncident()` only writes `incident_type` and `severity`. All other fields from `mapIncidentState` output (`erkennungszeitpunkt`, `erkannt_durch`, `erste_erkenntnisse`, `q1`, `q2`, `q3`, `playbook`, `regulatorische_meldungen`, `metadata`) are dropped before the Prisma write. The API schema (`src/api/schemas/incident.schema.ts`) also omits `erkennungszeitpunkt`, `erkannt_durch`, `erste_erkenntnisse` — so they're stripped by Zod before reaching the service.
- **Fix:** Update `CreateIncidentInputSchema` to accept all wizard fields; update `IncidentService.createIncident` to pass all fields to Prisma.

### GAP-3: MigrationService Cursor-Based Resume Never Wired
- **Affected:** DE5.1–DE5.4
- **Root cause:** `src/lib/migrationService.ts` (Phase 13) implements cursor-based resume, 30-day backup, and error classification. It is never imported. `src/hooks/useMigration.ts` (Phase 9) is the active implementation but restarts from index 0 on retry.
- **Fix:** Either wire `MigrationService` into `useMigration` hook, or accept that Phase 9's simpler migration is the implementation and document it as such. Dead code cleanup required.
