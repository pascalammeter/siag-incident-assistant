### 1. GitHub Secrets Not Configured (DE3.1–DE3.3)

- **Fix:** See `.github/workflows/README.md` — step-by-step setup instructions
- **Impact:** CI DB tests fail until configured; branch protection enforcement blocked

### 2. App Router Routes Have No @swagger Annotations (B6.1, B6.3)
- **Files:** `src/app/api/incidents/[id]/export/pdf/route.ts`, `src/app/api/swagger/route.ts`
- **Issue:** Swagger glob now scans `./src/app/api/**/*.ts` (fixed in 446df44) but none of these files have `@swagger` JSDoc blocks. PDF endpoint is absent from generated spec; spec documents POST for PDF (Express JSDoc) while implementation is GET.
- **Fix:** Add `@swagger` JSDoc to App Router PDF route documenting GET method and response schema
- **Impact:** Documentation only; API functions correctly

### 3. W1.1 Partial — Wizard Steps 1–5 Not API-Backed
- **File:** `src/components/wizard/WizardContext.tsx`
- **Issue:** createIncident() in Step6 is API-backed (W1.3 satisfied), but Steps 1–5 read/write WizardContext which is backed by localStorage. Full W1.1 compliance requires wizard state management to use useIncident() throughout.
- **Fix:** Refactor WizardContext to use useIncident() or thread incidentId through wizard state
- **Impact:** Functional — incident creation works; multi-session editing doesn't (W1.2 is the blocker)
