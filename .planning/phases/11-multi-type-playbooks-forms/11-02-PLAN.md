# Plan 11-02 — DDoS + Data Loss Playbooks

## Goal

Create DDoS and Data Loss incident playbooks (25 steps each) with type-specific content. Step 1 type selector includes DDoS and Data Loss options. Step 4 loads correct playbook based on type.

## Success Criteria

1. ✅ DDoS playbook (25 steps) covers detection, mitigation, upstream notification, communication
2. ✅ Data Loss playbook (25 steps) covers detection, containment, investigation, communication
3. ✅ Step 1 UI includes radio buttons for both DDoS and Data Loss types
4. ✅ Step 4 dynamically loads correct playbook (DDoS or Data Loss) based on incident_type
5. ✅ Playbook displays in table format with section headers
6. ✅ Type can be changed before Step 4 and playbook re-loads correctly
7. ✅ All existing tests still pass; 3+ new tests for DDoS/Data Loss loading
8. ✅ Playbook registry updated with all 4 types (Ransomware, Phishing, DDoS, Data Loss)

## Approach

### 1. Create DDoS Playbook (src/data/playbooks/ddos.ts)

25 steps in 4 sections:
- Detection (7 steps): traffic analysis, performance monitoring, alert responses
- Mitigation (7 steps): ISP notification, rate limiting, DDoS service activation
- Upstream Notification (4 steps): carrier contact, upstream provider communication
- Communication (7 steps): status pages, stakeholder updates, RTO tracking

### 2. Create Data Loss Playbook (src/data/playbooks/data-loss.ts)

25 steps in 4 sections:
- Detection (7 steps): data discovery, scope assessment, exfiltration indicators
- Containment (7 steps): access revocation, device isolation, account reset
- Investigation (7 steps): log collection, ransom note analysis, threat actor identification
- Communication (4 steps): stakeholder notification, ransom decision, law enforcement contact

### 3. Update Step 1 UI (src/app/page.tsx)

- Add "DDoS" and "Data Loss" radio buttons
- Update instructions to list all 5 types (Ransomware, Phishing, DDoS, Data Loss, Other)

### 4. Update Playbook Loader (src/app/step-4/page.tsx)

```typescript
import { ddosPlaybook } from '@/data/playbooks/ddos';
import { dataLossPlaybook } from '@/data/playbooks/data-loss';

// Add to switch/match:
case 'ddos': playbook = ddosPlaybook; break;
case 'data-loss': playbook = dataLossPlaybook; break;
```

### 5. Update Playbook Registry (src/data/playbooks.ts)

```typescript
export { ddosPlaybook } from './playbooks/ddos';
export { dataLossPlaybook } from './playbooks/data-loss';

export const getPlaybook = (type: string) => {
  switch(type) {
    case 'ransomware': return ransomwarePlaybook;
    case 'phishing': return phishingPlaybook;
    case 'ddos': return ddosPlaybook;
    case 'data-loss': return dataLossPlaybook;
    default: return genericPlaybook;
  }
}
```

### 6. Write Tests (src/__tests__/playbooks.test.ts)

- DDoS playbook has 25 steps, 4 sections, no invalid dependencies
- Data Loss playbook has 25 steps, 4 sections, no invalid dependencies
- Select DDoS in Step 1 → Step 4 loads DDoS playbook
- Select Data Loss in Step 1 → Step 4 loads Data Loss playbook
- Change type mid-workflow → playbook updates correctly
