## Phishing Playbook Content Strategy

Phase 1 (Erkennung — Detection):
- Technical analysis: Email headers, SPF/DKIM/DMARC validation
- Sender reputation checks: reverse DNS, domain registration
- URL/attachment analysis: sandboxing, detonation
- User reporting mechanism verification

Phase 2 (Eindaemmung — Containment):
- Isolate affected mailbox: disable access, preserve logs
- Revoke compromised credentials: password reset, 2FA removal/re-add
- Block attacker infrastructure: sender domain, malicious URLs, C2 IPs
- Prevent lateral movement: email forwarding rules, access controls

Phase 3 (Untersuchung — Investigation):
- Timeline reconstruction: when was phishing sent, when were credentials compromised, when was access first used
- Attacker artifacts: reply-to addresses, attachments (malware hashes), tracking pixels
- Identify compromised accounts: which users opened email, clicked links, entered credentials
- Scope assessment: how many users affected, what data accessed with compromised credentials

Phase 4 (Kommunikation — Communication):
- Internal notification: IT acknowledges to affected users
- User credential breach notification: password reset required for all affected
- Regulatory notification: if PII exposed (DSG/GDPR), report to authority
- External communication: customer notification if user data accessed (if customer-facing)

All content must be:
- Actionable (specific steps, not generic advice)
- Role-assigned (which team member performs each step)
- Realistic (based on NIST CSF, CISA phishing response guidance)
- Non-contradictory (avoid conflicting instructions)
