# Plan 11-01 — Phishing Playbook + Content Data Structure

## Goal

Establish the playbook data structure and populate Phishing incident type with 25-step playbook covering detection, containment, investigation, and communication. Users can select "Phishing" in Step 1 and see the Phishing playbook in Step 4.

## Success Criteria

1. ✅ Playbook TypeScript types created (interface Playbook, PlaybookStep, PlaybookSection)
2. ✅ Phishing playbook JSON/TS file contains 25 steps organized in 4 sections
3. ✅ Each step includes: number, title, action, responsible party, timeframe, dependencies
4. ✅ Step 1 UI includes radio button for "Phishing" incident type
5. ✅ Step 4 dynamically loads Phishing playbook when incident_type === 'phishing'
6. ✅ Playbook displays in table format with section headers
7. ✅ Type selection state persists across steps (via useIncident hook)
8. ✅ All tests passing (5+ playbook/type selection tests)

## Approach

### 1. Create Playbook Types (src/types/playbook.ts)

Define PlaybookStep, PlaybookSection, Playbook interfaces with:
- section: 'detection' | 'containment' | 'investigation' | 'communication'
- dependencies: step numbers for prerequisites
- title, action, responsible, timeframe fields

### 2. Create Phishing Playbook Data (src/data/playbooks/phishing.ts)

25 steps organized in 4 sections:
- Detection (7 steps): email analysis, sender verification, URL inspection
- Containment (7 steps): block email, lockdown accounts, reset credentials
- Investigation (7 steps): log collection, forensics, threat actor research
- Communication (4 steps): stakeholder updates, recovery timeline, training, review

### 3. Update Step 1 UI (src/app/page.tsx)

- Add "Phishing" radio button to type selector
- Update instructions to clarify type selection
- Maintain state via useIncident() hook

### 4. Update Step 4 Playbook Loader (src/app/step-4/page.tsx)

- Import phishingPlaybook
- Load correct playbook based on incident_type
- Display using PlaybookTable component (table format from Phase 6)

### 5. Create Playbook Registry (src/data/playbooks.ts)

Export all playbooks with loader function:
```typescript
export const getPlaybook = (type: string) => { ... }
```

### 6. Write Tests (src/__tests__/playbooks.test.ts)

- Phishing playbook structure validation
- Type selection in Step 1 → playbook loads in Step 4
- No circular dependencies in steps
- All 25 steps present and ordered correctly
