## Form Validation Patterns

### onBlur Validation in React Hook Form

From 11-RESEARCH.md Pattern 3:
```typescript
const form = useForm({
  resolver: zodResolver(schema),
  mode: 'onBlur',  // Trigger validation when field loses focus
  defaultValues: ...,
})
```

Current implementation (from StepForm.tsx):
- No mode specified → defaults to submit-only validation
- Need to add `mode: 'onBlur'` to all useForm() calls in form steps

### Error Styling

From 11-RESEARCH.md Pattern 4 & existing code:
```typescript
className={`border rounded-lg px-4 py-3 ${
  form.formState.errors.fieldName ? 'border-red-600 bg-red-50' : 'border-gray-300'
}`}
```

Required styling classes:
- `border-red-600` — red border on error
- `bg-red-50` — very light red background on error
- `text-red-600` — red text for error message
- Keep focus ring: `focus:ring-2 focus:ring-navy`

### Required Field Indicator

Current code shows asterisk in label:
```typescript
<label>
  Field Name <span className="text-red-600">*</span>
</label>
```

Verify all required fields have this indicator.

### Helper Text Pattern

From 11-RESEARCH.md Pattern 5:
Order of display:
1. Label with required indicator (*)
2. Input field
3. Helper text (if applicable) — gray text, smaller font, below input
4. Error message (if validation failed) — red text, overwrites helper on error

Helper text should be:
- Gray color: `text-gray-500`
- Smaller font: `text-xs`
- Below field with small margin: `mt-1`
- Non-required fields: show by default
- Required fields: show helper even if error present
