## Error Toast Integration

From research Pattern 6:
Current StepForm (lines 48-56) has try/catch but no error handling:
```typescript
const onSubmit = form.handleSubmit(async (data) => {
  setIsSubmitting(true)
  try {
    dispatch({ type: 'UPDATE_STEP', stepKey, data })
    dispatch({ type: 'NEXT_STEP' })
  } finally {
    setIsSubmitting(false)
  }
})
```

Enhancement needed:
- Wrap dispatch/NEXT_STEP in try/catch that catches API errors
- Import API_ERROR_MESSAGES from form-helpers
- Import toast from 'sonner'
- Map error code to message
- Show toast.error(message) on failure
- Still proceed to next step on success

Example:
```typescript
try {
  dispatch({ type: 'UPDATE_STEP', stepKey, data })
  dispatch({ type: 'NEXT_STEP' })
  toast.success('Schritt gespeichert.')
} catch (error) {
  const code = (error as any).code || 'SERVER_ERROR'
  const message = API_ERROR_MESSAGES[code] || API_ERROR_MESSAGES['SERVER_ERROR']
  toast.error(message)
}
```
