# Plan 11-04 — Error Mapping + Toast Integration

## Goal

Map technical API errors to user-friendly messages and integrate toast notifications for network/API errors, save feedback, and loading states. Save button shows loading spinner and is disabled during API request.

## Success Criteria

1. Zod validation errors mapped to user-friendly messages ("Field is required" not "String.required")
2. API validation errors (400) mapped to field-level error messages or general toast
3. Network errors and API failures trigger toast notification with message and retry option
4. Save button shows loading spinner during API request
5. Save button disabled to prevent double-submit during API request
6. Toast component supports light/dark modes (respects theme from Phase 10)
7. Toast displays for 4-5 seconds then auto-dismisses
8. Toast position: bottom-right or top-right, accessible (ARIA labels)
9. Error message mapping documented (constants/errorMessages.ts)
10. 70%+ test coverage on error mapping and toast behavior

## Approach

### 1. Create Error Message Mapping (src/constants/errorMessages.ts)

Map Zod error codes and API response codes to user-friendly messages.
- Zod: String.email, String.min, String.max, Array.min, Date.invalid
- API: INCIDENT_NOT_FOUND, UNAUTHORIZED, FORBIDDEN, CONFLICT, INTERNAL_SERVER_ERROR
- Network: NETWORK_ERROR, TIMEOUT

### 2. Create Toast Component (src/components/Toast.tsx)

Toast displays message with type (success/error/warning/info), auto-dismisses after 4-5 seconds.
Optional action button for "Retry" scenarios. Dismiss button always available.
Respects light/dark mode via Tailwind theme classes.

### 3. Create Toast Context (src/context/ToastContext.tsx)

Context provides showToast(message, type, action) and dismissToast(id) functions.
Wrap root layout with ToastProvider. useToast() hook for consuming components.

### 4. Update API Error Handling (src/services/api.ts)

Catch API errors (response not ok) and throw mapped error objects with code and details.
Catch network errors (TypeError) separately.
Include error code so handlers can map to user message.

### 5. Update StepForm with Save Button + Loading State

Add isLoading state. Show spinner and disable button during API request.
On success: show success toast, proceed to next step.
On error: show error toast with Retry action (optional).

### 6. Dark Mode Support for Toast

Use Tailwind dark: prefix for all toast styling (text, background, borders).
Test in both light and dark modes to ensure readability.

### 7. Write Tests (src/__tests__/errors.test.ts)

Unit tests for error message mapping.
Integration tests for API error handling.
Component tests for Toast (render, dismiss, timeout, dark mode).
Form tests for save button state (disabled during request, spinner displays).
