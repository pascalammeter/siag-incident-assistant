# Phase 3 — UI Design Contract

> Visual and interaction contract for Screens 0-3 (Triage). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (pure Tailwind v4) |
| Preset | not applicable |
| Component library | none — hand-built with Tailwind utility classes |
| Icon library | none — use Unicode/emoji sparingly (checkmark, warning) |
| Font | Inter, ui-sans-serif, system-ui, sans-serif (declared in globals.css @theme) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | gap-1, p-1 | Icon gaps, inline padding |
| sm | 8px | gap-2, p-2, mb-2 | Compact element spacing, label margins |
| md | 16px | gap-4, p-4, mb-4 | Default element spacing, card padding |
| lg | 24px | gap-6, p-6, space-y-6 | Section padding, step content spacing |
| xl | 32px | gap-8, p-8 | Layout gaps, hero section padding |
| 2xl | 48px | py-12 | Major section breaks |
| 3xl | 64px | py-16 | Page-level vertical spacing |

Exceptions:
- Touch targets: `min-h-[44px]` on all buttons and interactive elements (NF1.4)
- Checkbox: `w-5 h-5` (20px) — established in StepInterstitial placeholder

---

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|-----------------|
| Body | 16px | 400 (normal) | 1.5 | `text-base font-normal leading-relaxed` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Caption | 12px | 400 (normal) | 1.4 | `text-xs` |
| Heading (step) | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |
| Display (hero) | 32px | 700 (bold) | 1.2 | `text-3xl font-bold` |

Notes:
- Max 2 weights used: 400 (normal body/caption) and 700 (bold headings/CTAs). 500 (medium) only for labels and button text.
- All text inherits `color: var(--color-navy)` from body unless overridden.

---

## Color

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | #ffffff white | `bg-white` | Page background, card backgrounds |
| Secondary (30%) | #f5f7fa lightgray | `bg-lightgray` | Step content area, input backgrounds, hover states |
| Primary (nav/CTA) | #1a2e4a navy | `bg-navy text-navy` | Buttons, headings, progress bar, borders |
| Warning (10%) | #f59e0b amber | `bg-amber text-amber border-amber` | No-Go rules, Meldefrist banner, Eskalation alert |
| Destructive | #dc2626 red-600 | `text-red-600 border-red-600` | Form validation errors only — NOT for UI chrome |

### Color application rules:
- **Amber reserved for:** No-Go rule cards (Screen 0), Meldefrist banner (Screen 2), Eskalation alert bei Kritisch (Screen 3), No-Go inline warnings
- **Navy reserved for:** Primary CTA buttons, step headings, progress bar fill, text body color
- **Red reserved for:** Zod validation error messages and error ring on inputs. Never as background. Never as alert styling (use amber instead per NF2.2).
- **Amber backgrounds:** Use `bg-amber/10` (10% opacity) with `border-l-4 border-amber` for warning cards. Never full `bg-amber` as background.

## Component Inventory — Screen 0 (StepInterstitial)

### Layout
- Full-width within wizard shell (`max-w-4xl` inherited from layout)
- Vertical stack: heading > intro text > 8 rule cards > checkbox > confirm button
- Spacing between sections: `space-y-6` (24px)

### No-Go Rule Card (x8)
- Container: `border-l-4 border-amber bg-amber/10 rounded-r-lg p-4`
- Icon: Unicode warning triangle (U+26A0) or amber dot, left-aligned
- Text: `text-sm font-medium text-navy`
- Each rule is a separate card, not a list

### 8 No-Go Rules — Exact Copy
| # | Rule |
|---|------|
| 1 | Systeme nicht neu starten — Beweise gehen verloren |
| 2 | Kein Loesegeld zahlen — keine Garantie fuer Entschluesselung |
| 3 | Keinen Kontakt mit Angreifern ohne juristischen Beistand |
| 4 | Keine Bereinigung vor der forensischen Sicherung |
| 5 | Keine Logdateien loeschen oder ueberschreiben |
| 6 | Vorfall nicht vertuschen — Meldepflichten beachten |
| 7 | Backups sofort isolieren — nicht auf infizierte Systeme zugreifen |
| 8 | Normale Arbeit sofort stoppen — Schadensbegrenzung priorisieren |

### Confirmation Checkbox
- Container: `flex items-start gap-3 cursor-pointer` (items-start for multi-line label alignment)
- Input: `w-5 h-5 rounded border-gray-300 accent-navy`
- Label: `text-sm text-gray-700` — "Ich habe diese Hinweise gelesen und verstanden."

### Confirm Button
- Style: `bg-navy text-white px-6 py-3 rounded-lg font-medium min-h-[44px]`
- Disabled state: `opacity-50 cursor-not-allowed` when checkbox unchecked
- Label: **"Verstanden — Weiter"**
- Full width on mobile: `w-full sm:w-auto`

### Interaction Contract
- Button disabled until checkbox checked
- On confirm: dispatch `CONFIRM_NO_GO` + `NEXT_STEP`
- No StepNavigator shown on this screen (Interstitial uses own button)
- localStorage flag `noGoConfirmed` persisted via wizard state

### Layout
- Centered content, vertical stack
- Hero section with large CTA in center

## Component Inventory — Screen 2 (Step2Erfassen)

### Layout
- Vertical form stack within StepForm wrapper
- Spacing between form groups: `space-y-6`
- Each form group: label + input + error message

### Meldefrist Banner (top of screen, before form)
- Container: `border-l-4 border-amber bg-amber/10 rounded-r-lg p-4 flex items-center gap-3`
- Icon: Clock/warning icon (Unicode U+23F0 or U+26A0)
- Copy: **"Achtung: Mit der Erkennung eines Vorfalls beginnen rechtliche Meldefristen zu laufen."**
- Style: `text-sm font-medium text-navy`

### Form Field Pattern (reusable)
- Label: `text-sm font-medium text-navy mb-1 block`
- Input: `w-full border border-gray-300 rounded-lg px-4 py-3 text-base bg-white focus:ring-2 focus:ring-navy focus:border-navy outline-none min-h-[44px]`
- Error: `text-sm text-red-600 mt-1` — shown only after Zod validation fails
- Required indicator: `text-red-600` asterisk after label text

### Field 1: Erkennungszeitpunkt (Datum + Uhrzeit)
- Type: `datetime-local` input
- Label: **"Wann wurde der Vorfall erkannt? *"**
- "Jetzt"-Button: `bg-lightgray border border-navy text-navy px-4 py-2 rounded-lg text-sm font-medium min-h-[44px]` — inline next to input
- Layout: `flex gap-2 items-end` (input + button side by side)
- Button label: **"Jetzt"**
- Auto-fills current date+time on click

### Field 2: Erkannt durch
- Type: Select dropdown
- Label: **"Durch wen wurde der Vorfall erkannt? *"**
- Options: "IT-Mitarbeiter" | "Nutzer/Mitarbeitende" | "Externes System/Monitoring" | "Angreifer-Kontakt" | "Sonstiges"
- Style: same as input pattern with `appearance-none` and custom chevron

### Field 3: Betroffene Systeme
- Type: Multi-select checkboxes (not a <select multiple>)
- Label: **"Betroffene Systeme/Assets"**
- Options as checkbox group, each: `flex items-center gap-2`
- Checkbox: `w-5 h-5 rounded border-gray-300 accent-navy`
- Checkbox label: `text-sm text-navy`
- Options: "Workstations" | "Server" | "Backups" | "E-Mail" | "Netzwerk" | "OT/ICS" | "Sonstiges"
- Layout: 2-column grid on desktop `grid grid-cols-1 sm:grid-cols-2 gap-3`

### Field 4: Erste Auffaelligkeiten
- Type: Textarea
- Label: **"Erste Auffaelligkeiten / Beschreibung"**
- Style: same as input pattern but `min-h-[120px] resize-y`
- Placeholder: "Was ist Ihnen aufgefallen? Z.B. verschluesselte Dateien, ungewoehnliche Netzwerkaktivitaet..."

### Field 5: Ransomware-Checkbox
- Type: Single checkbox
- Container: `border-l-4 border-amber bg-amber/10 rounded-r-lg p-4 flex items-start gap-3`
- Checkbox: `w-5 h-5 rounded border-gray-300 accent-navy mt-0.5`
- Label: **"Ist ein Loesegeld-Schreiben oder eine Verschluesselungs-Meldung vorhanden?"**
- Style: `text-sm font-medium text-navy`

### Timestamp Display (after form, above navigation)
- Shown only after Erkennungszeitpunkt is filled
- Container: `bg-navy text-white rounded-lg p-4 text-center`
- Heading: `text-sm font-medium opacity-80` — "Erkennungszeitpunkt"
- Time: `text-xl font-bold` — formatted timestamp
- Subtext: `text-sm opacity-80 mt-1` — **"Ihre rechtliche Meldefrist beginnt jetzt."**

### Interaction Contract
- All fields managed by react-hook-form + Zod
- Required fields: Erkennungszeitpunkt, Erkannt-durch
- Optional fields: Betroffene Systeme, Auffaelligkeiten, Ransomware-Checkbox
- Validation on "Weiter" button click (not on blur for stress-UX)
- StepNavigator shown with "Weiter" / "Zurueck"

## Component Inventory — Screen 3 (Step3Klassifikation)

### Layout
- Vertical stack: heading > decision tree > severity result > incident type > escalation alert
- Spacing: `space-y-6`

### Decision Tree (3 binary questions)
- Each question in its own card: `bg-lightgray rounded-lg p-6`
- Question text: `text-base font-medium text-navy mb-4`
- Answer buttons: pill-style toggle group, horizontal

#### Question Card Pattern
- Question number: `text-xs font-medium text-gray-500 mb-1` — "Frage 1 von 3"
- Question text: `text-base font-bold text-navy mb-4`
- Button group: `flex gap-3`
- Active button: `bg-navy text-white px-6 py-3 rounded-lg font-medium min-h-[44px]`
- Inactive button: `bg-white border border-gray-300 text-navy px-6 py-3 rounded-lg font-medium min-h-[44px] hover:border-navy transition-colors`

#### Question 1
- **"Sind kritische Geschaeftssysteme betroffen oder verschluesselt?"**
- Options: **"Ja"** | **"Nein"**

#### Question 2
- **"Sind personenbezogene Daten betroffen oder potenziell abgeflossen?"**
- Options: **"Ja"** | **"Nein"**

#### Question 3
- **"Ist der Angreifer noch aktiv im Netzwerk?"**
- Options: **"Ja"** | **"Nein"** | **"Unbekannt"**

### Severity Result (auto-calculated)
- Shown only after all 3 questions answered
- Container depends on severity:

#### Kritisch (any "Ja" on Q1 or Q3="Ja")
- `border-l-4 border-amber bg-amber/10 rounded-r-lg p-6`
- Badge: `inline-block bg-amber text-white text-xs font-bold px-3 py-1 rounded-full uppercase` — "KRITISCH"
- Text: `text-base font-medium text-navy mt-2` — "Sofortige Eskalation erforderlich. SIAG-Berater einbeziehen."

#### Hoch (Q2="Ja" but Q1="Nein" and Q3!="Ja")
- `border-l-4 border-navy bg-lightgray rounded-r-lg p-6`
- Badge: `inline-block bg-navy text-white text-xs font-bold px-3 py-1 rounded-full uppercase` — "HOCH"
- Text: `text-base font-medium text-navy mt-2` — "Erhoehte Aufmerksamkeit. Datenschutzrelevanz pruefen."

#### Mittel (all "Nein")
- `border-l-4 border-gray-400 bg-lightgray rounded-r-lg p-6`
- Badge: `inline-block bg-gray-500 text-white text-xs font-bold px-3 py-1 rounded-full uppercase` — "MITTEL"
- Text: `text-base font-medium text-navy mt-2` — "Standardprozess. Dokumentation und Beobachtung."

### Severity Logic
```
if Q1 === "Ja" || Q3 === "Ja" → KRITISCH
else if Q2 === "Ja" → HOCH
else → MITTEL
```
Note: Q3 "Unbekannt" counts as "Nein" for severity but triggers note: "Unbekannt wird als potenziell aktiv behandelt — erhoehte Wachsamkeit."

Wait — correction per CISO best practice: Q3 "Unbekannt" should escalate to KRITISCH (assume worst case).
```
if Q1 === "Ja" || Q3 === "Ja" || Q3 === "Unbekannt" → KRITISCH
else if Q2 === "Ja" → HOCH
else → MITTEL
```

### Incident-Typ Auswahl
- Label: **"Incident-Typ"**
- Type: Radio button group, vertical
- Options: "Ransomware" | "Phishing" | "DDoS" | "Datenverlust" | "Unbefugter Zugriff" | "Sonstiges"
- Radio style: `flex items-center gap-3 p-3 rounded-lg border border-gray-200 hover:border-navy cursor-pointer`
- Selected: `border-navy bg-lightgray`
- Radio input: `w-5 h-5 accent-navy`
- Default pre-selected: "Ransomware" (MVP primary path per F5.3)

### Eskalation Alert (Kritisch only)
- Shown only when severity = KRITISCH
- Container: `border-2 border-amber bg-amber/10 rounded-lg p-6 text-center`
- Icon: Unicode warning U+26A0 large `text-3xl`
- Heading: `text-lg font-bold text-navy mt-2` — **"Schweregrad Kritisch"**
- Body: `text-base text-navy mt-1` — **"SIAG-Berater sofort einbeziehen. Eskalieren Sie umgehend an die Geschaeftsleitung."**
- CTA hint: `text-sm text-gray-500 mt-3` — "Der Kontakt zu SIAG erfolgt im letzten Schritt."

### Interaction Contract
- Decision tree questions managed by react-hook-form + Zod
- All 3 questions required before "Weiter" is enabled
- Incident-Typ required (defaults to Ransomware)
- Severity calculated reactively as answers change
- StepNavigator shown with "Weiter" / "Zurueck"
