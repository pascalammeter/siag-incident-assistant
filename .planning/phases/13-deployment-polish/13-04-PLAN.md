# Phase 13-04 Plan: UAT & Consultant Sign-off

## Overview
Conduct comprehensive UAT with SIAG incident response consultant. Verify all 4 incident types (Ransomware, Phishing, DDoS, Data Loss) work end-to-end with correct playbooks, test PDF export visual quality and compliance, verify performance on mobile networks, and obtain written sign-off confirming v1.1 is production-ready.

## Success Criteria (MUST be TRUE)
1. ✅ SIAG consultant executes all 4 incident type workflows end-to-end; no blockers encountered
2. ✅ PDF export verified for professional appearance (logo, metadata, proper formatting, readable on print)
3. ✅ Performance tested on simulated slow mobile network (3G); app remains responsive
4. ✅ Compliance logic verified correct (ISG 24h, DSG, FINMA 24/72h deadlines calculated accurately)
5. ✅ Incident list and resume functionality tested and approved
6. ✅ Dark mode and accessibility features tested; no contrast issues
7. ✅ SIAG consultant provides written sign-off email confirming production readiness
8. ✅ Sign-off documented in project record (e.g., SIGN-OFF.md)

## Tasks

### 1. UAT Environment Setup (1.5 hours)
- [ ] Create UAT test account for SIAG consultant with production data isolation
- [ ] Deploy v1.1 to staging/preview URL (if not using production)
- [ ] Prepare test data: sample incidents from v1.0 (if available) for resume testing
- [ ] Document staging URL and access instructions
- [ ] Send consultant email with:
  - UAT URL and login credentials
  - UAT timeline (2-3 hours recommended)
  - Feature overview (new in v1.1: API-backed storage, dark mode, PDF export, multi-type support)
  - Test plan/checklist (below)
  - Feedback collection method (form, email, call)

### 2. Test Plan & Checklist Preparation (1 hour)
- [ ] Create detailed UAT checklist document covering:

#### Ransomware Incident (Type 1)
- [ ] Step 1: Select "Ransomware" incident type
- [ ] Step 2: Enter ransom note details
- [ ] Step 3: Fill personal data fields (name, email, phone)
- [ ] Step 4: Review Ransomware playbook (25 steps)
- [ ] Step 5: Fill response details
- [ ] Step 6: Review summary and verify all data captured
- [ ] **Test PDF Export:** Click "Export as PDF" → verify document contains:
  - [ ] SIAG logo and branding
  - [ ] Incident type (Ransomware)
  - [ ] All steps and responses from wizard
  - [ ] Compliance deadlines (ISG 24h, DSG)
  - [ ] Professional formatting, readable on print
- [ ] **Test Compliance:** Verify ISG deadline = now + 24h, DSG deadline = now + 15 days
- [ ] **Resume Test:** Create but don't submit; reload browser → incident appears in list; click to resume → wizard hydrates correctly

#### Phishing Incident (Type 2)
- [ ] Repeat above workflow with Phishing type
- [ ] **Phishing-Specific Test:**
  - [ ] Verify phishing playbook contains detection/containment/communication steps
  - [ ] Test email addressing fields (sender, recipient, subject)
  - [ ] Verify screenshot upload field (if present)
  - [ ] PDF export includes phishing URLs/indicators correctly

#### DDoS Incident (Type 3)
- [ ] Repeat workflow with DDoS type
- [ ] **DDoS-Specific Test:**
  - [ ] Verify playbook includes mitigation, upstream notification steps
  - [ ] Test attack characteristics fields (source IPs, attack type)
  - [ ] Verify PDF includes network details

#### Data Loss Incident (Type 4)
- [ ] Repeat workflow with Data Loss type
- [ ] **Data Loss-Specific Test:**
  - [ ] Verify playbook includes data classification, notification, legal steps
  - [ ] Test affected_systems field (checkboxes or multi-select)
  - [ ] Verify FINMA deadline calculation (24-72h depending on severity)
  - [ ] PDF includes data loss indicators

### 3. Mobile & Accessibility Testing (1.5 hours)

#### Mobile Performance
- [ ] Open v1.1 on mobile device (iPhone or Android)
- [ ] Test all wizard steps on small screen:
  - [ ] Steps are readable (no horizontal scroll needed)
  - [ ] Buttons are tap-friendly (min 44px height)
  - [ ] Playbook scrolls smoothly without lag
  - [ ] Dark mode toggle works on mobile
  - [ ] PDF export works from mobile browser
- [ ] Simulate slow network using Chrome DevTools throttling (3G):
  - [ ] App loads in <5 seconds
  - [ ] API calls complete within <10 seconds (show spinner)
  - [ ] PDF export initiates but takes longer (acceptable)
  - [ ] Error messages appear if network fails mid-request

#### Accessibility
- [ ] Test keyboard navigation (Tab key):
  - [ ] All form fields accessible via Tab
  - [ ] Buttons focusable and clickable with Enter
  - [ ] Focus indicators visible
- [ ] Test screen reader (VoiceOver on Mac, NVDA on Windows):
  - [ ] Page title read correctly
  - [ ] Form labels associated with inputs
  - [ ] Error messages announced
- [ ] Check color contrast (use WebAIM Contrast Checker):
  - [ ] All text ≥4.5:1 ratio on light background
  - [ ] Dark mode text ≥4.5:1 ratio on dark background

### 4. Consultant Feedback Collection (1-2 hours)
