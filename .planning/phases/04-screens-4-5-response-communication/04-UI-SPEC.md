# Phase 4 — UI Design Contract

> Visual and interaction contract for Screen 4 (Reaktionsschritte) and Screen 5 (Kommunikation & Eskalation). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (native HTML + Tailwind v4) |
| Icon library | none (Unicode entities: &#9888; for warnings, &#10003; for checks) |
| Font | Inter, ui-sans-serif, system-ui, sans-serif (via `--font-sans` in globals.css @theme) |

Source: globals.css @theme block, established in Phase 1.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`), amber box padding |
| lg | 24px | Section padding (`p-6`, `space-y-6`) |
| xl | 32px | Layout gaps (`space-y-8`) between phase groups on Screen 4 |
| 2xl | 48px | Not used this phase |
| 3xl | 64px | Not used this phase |

Exceptions:
- `min-h-[44px]` on all interactive elements (checkboxes rows, buttons, textareas min-height) per NF1.4 touch target requirement.
- `pt-6` on StepNavigator bar (established pattern).

---

## Typography

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Body | 16px | 400 (normal) | 1.5 | `text-base font-normal leading-relaxed` |
| Label | 14px | 400 (normal) | 1.5 | `text-sm font-normal` |
| Heading (screen) | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |
| Subheading (phase group) | 18px | 700 (bold) | 1.3 | `text-lg font-bold` |

Source: Established in Phase 3 screens. `text-2xl font-bold text-navy` for screen headings, `text-sm font-normal` for labels, `text-base` for body content.

Additional rules:
- Phase group titles on Screen 4 (Sofortmassnahmen, Eindaemmung, etc.) use Subheading role.
- Progress counter "X von 25 erledigt" uses Label role with `font-bold`.
- Deadline display "Frist bis: ..." uses Body role with `font-bold`.
- Placeholder text in textareas: `text-sm text-gray-400`.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (white) | Page background, form surfaces |
| Secondary (30%) | `#f5f7fa` (lightgray) | Phase group cards on Screen 4, question cards on Screen 5, textarea backgrounds |
| Accent (10%) | `#1a2e4a` (navy) | See reserved-for list below |
| Warning | `#f59e0b` (amber) | No-Go inline boxes, deadline urgency indicator |

Accent reserved for:
- Screen heading text (`text-navy`)
- Primary CTA buttons (`bg-navy text-white`)
- Checkbox accent color (`accent-navy`)
- Role badge background (`bg-navy text-white` and variants)
- SIAG CTA button (`bg-navy text-white`)
- Focus rings (`focus:ring-navy`)
- Active/selected state borders (`border-navy`)

Additional semantic colors (established patterns):
- `text-red-600`: Validation error messages, required field asterisks
- `text-gray-600`: Introductory paragraph text
- `text-gray-500`: Secondary metadata (e.g., "Frage X von Y" counter pattern)
- `text-gray-400`: Placeholder text
- `opacity-50 cursor-not-allowed`: Disabled button state (on `bg-navy`)
- `bg-navy text-white`: Erkennungszeitpunkt display banner (reused from Screen 2 for deadline cards on Screen 5)

## Component Inventory — Screen 4 (Reaktionsschritte)

### S4-01: Screen Header
```
<h2 class="text-2xl font-bold text-navy">Reaktionsschritte</h2>
<p class="text-base text-gray-600 leading-relaxed">
  Arbeiten Sie die Ransomware-Checkliste Schritt fuer Schritt ab. Alle 25 Punkte muessen bestaetigt werden.
</p>
```

### S4-02: Progress Counter
```
<div class="bg-lightgray rounded-lg p-4 flex items-center justify-between">
  <span class="text-sm font-bold text-navy">{completedCount} von 25 erledigt</span>
  <div class="w-48 h-2 bg-gray-200 rounded-full overflow-hidden">
    <div class="h-full bg-navy rounded-full transition-all" style="width: {percent}%" />
  </div>
</div>
```
- Progress bar: `bg-gray-200` track, `bg-navy` fill, `h-2` height, `rounded-full`.
- Updates reactively as checkboxes are toggled.

### S4-03: Phase Group
```
<div class="space-y-3">
  <h3 class="text-lg font-bold text-navy">Phase 1: Sofortmassnahmen</h3>
  <!-- checklist items -->
</div>
```
- 4 groups separated by `space-y-8` between groups.
- Phase titles: `text-lg font-bold text-navy`.
- Items within a group: `space-y-3`.

### S4-04: Checklist Item (standard)
```
<label class="flex items-start gap-3 p-4 bg-lightgray rounded-lg cursor-pointer hover:bg-gray-100 transition-colors">
  <input type="checkbox" class="w-5 h-5 accent-navy mt-0.5 shrink-0" />
  <div class="flex-1">
    <span class="text-sm text-navy">{step text}</span>
    <span class="inline-block ml-2 text-xs font-bold bg-navy/10 text-navy px-2 py-0.5 rounded-full">{role}</span>
  </div>
</label>
```
- Touch target: entire row is clickable via `<label>` wrapping the checkbox.
- Checked state: `bg-lightgray` stays the same (no dramatic color change — stress-appropriate).
- Checkbox: `w-5 h-5 accent-navy` (established pattern from Interstitial and Screen 2).

### S4-05: Checklist Item (with No-Go inline warning)
```
<div class="space-y-0">
  <label class="flex items-start gap-3 p-4 bg-lightgray rounded-t-lg cursor-pointer">
    <input type="checkbox" class="w-5 h-5 accent-navy mt-0.5 shrink-0" />
    <div class="flex-1">
      <span class="text-sm text-navy">{step text}</span>
      <span class="inline-block ml-2 text-xs font-bold bg-navy/10 text-navy px-2 py-0.5 rounded-full">{role}</span>
    </div>
  </label>
  <div class="border-l-4 border-amber bg-amber/10 rounded-br-lg p-4">
    <p class="text-sm font-normal text-navy">
      <span class="mr-2">&#9888;</span>
      {no-go warning text}
    </p>
  </div>
</div>
```
- No-Go box uses identical pattern from StepInterstitial: `border-l-4 border-amber bg-amber/10 rounded-r-lg p-4`.
- Attached directly below its parent checklist item with `rounded-t-lg` on item and `rounded-br-lg` on warning for visual connection.

### S4-06: Role Badge
```
<span class="inline-block text-xs font-bold bg-navy/10 text-navy px-2 py-0.5 rounded-full">{role}</span>
```
Roles and their display:
- `IT-Leiter` — default badge (navy/10 background)
- `CISO` — default badge
- `CEO` — default badge
- `Forensik` — default badge

All role badges use the same single style. No color differentiation between roles — keeps the UI calm and avoids cognitive overload during incident stress.

### S4-07: Disabled "Weiter" Button
The StepNavigator already supports `isNextDisabled` prop (confirmed in StepNavigator.tsx):
```
<StepNavigator isNextDisabled={completedCount < 25} ... />
```
Disabled state: `opacity-50 cursor-not-allowed` on `bg-navy` button (established pattern).

## Component Inventory — Screen 5 (Kommunikation & Eskalation)

### S5-01: Screen Header
```
<h2 class="text-2xl font-bold text-navy">Kommunikation & Eskalation</h2>
<p class="text-base text-gray-600 leading-relaxed">
  Pruefen Sie Ihre Meldepflichten und bereiten Sie die Kommunikation vor.
</p>
```

### S5-02: Meldepflicht Question Card
```
<div class="bg-lightgray rounded-lg p-6 space-y-3">
  <p class="text-base font-bold text-navy">{question text}</p>
  <div class="flex gap-3">
    <button type="button" class="{selected: bg-navy text-white | unselected: bg-white border border-gray-300 text-navy hover:border-navy} px-6 py-3 rounded-lg font-normal min-h-[44px]">
      Ja
    </button>
    <button type="button" class="... min-h-[44px]">
      Nein
    </button>
  </div>
</div>
```
- Same pill button pattern as Screen 3 Klassifikation questions.
- 3 cards stacked with `space-y-4`.

### S5-03: Deadline Display Card (conditional, shown per triggered obligation)
```
<div class="bg-navy text-white rounded-lg p-4">
  <div class="flex items-center justify-between">
    <div>
      <p class="text-sm font-normal opacity-80">{law reference, e.g. "ISG — NCSC-Meldung"}</p>
      <p class="text-base font-bold">Frist bis: {formatted deadline}</p>
    </div>
    <span class="text-xs font-bold bg-white/20 text-white px-3 py-1 rounded-full">{duration label, e.g. "24h"}</span>
  </div>
</div>
```
- Reuses the `bg-navy text-white rounded-lg p-4` pattern from Screen 2 Erkennungszeitpunkt display.
- Static deadline (D-06): formatted as "Freitag, 04. Apr 2026 14:32 Uhr" using `toLocaleString('de-CH', { weekday: 'long', day: '2-digit', month: 'short', year: 'numeric', hour: '2-digit', minute: '2-digit' })`.
- Multiple deadline cards stacked with `space-y-3`.
- Only shown when corresponding question answered "Ja".

### S5-04: DSG Special Case Display
```
<div class="bg-navy text-white rounded-lg p-4">
  <p class="text-sm font-normal opacity-80">DSG/DSGVO — FDPIC-Meldung</p>
  <p class="text-base font-bold">So schnell wie moeglich</p>
  <p class="text-sm opacity-80 mt-1">Keine feste Frist — unverzueglich nach Kenntnis.</p>
</div>
```
- Same card style but no computed deadline (DSG has no fixed hour count).

### S5-05: Kommunikations-Checkliste
```
<div class="space-y-3">
  <h3 class="text-lg font-bold text-navy">Kommunikations-Checkliste</h3>
  {items.map(item => (
    <label class="flex items-center gap-3 p-3 bg-lightgray rounded-lg cursor-pointer">
      <input type="checkbox" class="w-5 h-5 accent-navy shrink-0" />
      <span class="text-sm text-navy">{item}</span>
    </label>
  ))}
</div>
```
Items (in order):
1. Krisenstab informiert
2. Geschaeftsleitung + Verwaltungsrat informiert
3. Mitarbeitende informiert
4. Medien/Oeffentlichkeit (falls erforderlich)
5. Kunden informiert (falls betroffen)
6. Partner/Lieferanten informiert (falls betroffen)

### S5-06: Kommunikationsbaustein (Text Template)
```
<div class="border border-gray-200 rounded-lg overflow-hidden">
  <div class="bg-lightgray px-4 py-3 flex items-center justify-between border-b border-gray-200">
    <h4 class="text-sm font-bold text-navy">{template title}</h4>
    <button type="button" class="text-sm font-bold text-navy bg-white border border-navy px-4 py-2 rounded-lg min-h-[44px] hover:bg-lightgray transition-colors">
      Kopieren
    </button>
  </div>
  <textarea class="w-full px-4 py-3 text-sm text-navy bg-white border-0 outline-none focus:ring-2 focus:ring-navy min-h-[200px] resize-y" />
</div>
```
- 3 templates stacked with `space-y-6`.
- Template titles: "Vorlage: Geschaeftsleitung / VR", "Vorlage: Mitarbeitende", "Vorlage: Medien / Oeffentlichkeit".
- Textarea pre-filled with template text containing dynamic data from wizard state and `[Platzhalter]` brackets for unknown data.
- "Kopieren" button uses Clipboard API (`navigator.clipboard.writeText()`).

### S5-07: Kopieren Button — Success Feedback
```
<button class="text-sm font-bold text-white bg-navy px-4 py-2 rounded-lg min-h-[44px]">
  Kopiert &#10003;
</button>
```
- On click: button text changes from "Kopieren" to "Kopiert" with checkmark for 2 seconds, then reverts.
- Background changes to solid `bg-navy text-white` during success state.
- No toast/snackbar — inline feedback only (stress-appropriate, minimal distraction).

### S5-08: SIAG CTA Block
```
<div class="border-2 border-navy bg-lightgray rounded-lg p-6 text-center space-y-4">
  <p class="text-lg font-bold text-navy">SIAG-Berater jetzt einbeziehen</p>
  <p class="text-base text-gray-600">
    Unsere Incident-Response-Spezialisten unterstuetzen Sie bei der Bewaeltigung.
  </p>
  <div class="bg-white rounded-lg p-4 inline-block text-left space-y-2">
    <p class="text-sm text-navy"><span class="font-bold">Telefon:</span> +41 XX XXX XX XX</p>
    <p class="text-sm text-navy"><span class="font-bold">E-Mail:</span> incident@siag.ch</p>
  </div>
  <p class="text-xs text-gray-400">Kontaktdaten werden vor Go-Live aktualisiert.</p>
</div>
```
- Static display (D-09): no mailto link, no tel link.
- Placeholder contact info for review phase.
- Positioned after Kommunikationsbausteine, before StepNavigator.

### Template Placeholder Tokens

Dynamic (from wizard state):
- `{erkennungszeitpunkt}` — formatted date from `state.erfassen.erkennungszeitpunkt`
- `{severity}` — "KRITISCH", "HOCH", or "MITTEL" from `state.klassifikation.severity`
- `{incidentType}` — "Ransomware" etc. from `state.klassifikation.incidentType`
- `{betroffene_systeme}` — comma-joined list from `state.erfassen.betroffene_systeme`

Static placeholders (user must replace):
- `[Firmenname]`
- `[Name des Ansprechpartners]`
- `[Ihre E-Mail]`
- `[Telefonnummer]`

### Empty / Edge States

| State | Copy |
|-------|------|
| Screen 4 — 0 of 25 checked | Progress counter shows "0 von 25 erledigt", progress bar at 0% |
| Screen 5 — no Meldepflicht questions answered | Deadline cards hidden. Section shows only the 3 question cards. |

### Error States

| Error | Copy |
|-------|------|
| Clipboard API fails | Button text changes to "Kopieren fehlgeschlagen" for 2 seconds, then reverts to "Kopieren". No toast. |
| erkennungszeitpunkt missing for deadline calc | Amber warning: "Erkennungszeitpunkt fehlt — Fristen koennen nicht berechnet werden. Gehen Sie zurueck zu Schritt 2." |

### Destructive Actions

None in this phase. No data deletion, no reset functionality. All checkbox toggling is reversible.

## Interaction Contracts

### Screen 4

| Interaction | Behavior |
|-------------|----------|
| Checkbox toggle | Toggles step ID in `completedSteps` array in ReaktionData. Counter updates immediately. |
| Scroll position | No auto-scroll on check. User controls their own pace. |
| "Weiter" disabled state | `opacity-50 cursor-not-allowed` until `completedSteps.length === 25` |
| "Zurueck" button | Always enabled. Navigates to Screen 3. No confirmation needed. |
| Page reload | `completedSteps` persisted via localStorage (wizard infrastructure). Checkboxes restored on reload. |

### Screen 5

| Interaction | Behavior |
|-------------|----------|
| Meldepflicht Ja/Nein toggle | Pill button selection (same as Screen 3). Corresponding deadline card appears/disappears with no animation. |
| Kommunikations-Checkliste toggle | Simple checkbox toggle. State stored in KommunikationData. |
| Textarea editing | User can freely edit pre-filled template text. Changes persist in component state. |
| "Kopieren" button | Copies current textarea content to clipboard. 2-second inline success feedback. |
| "Weiter" button | Always enabled (Screen 5 has no completion gate — D-03 only applies to Screen 4). |
| Page reload | Meldepflicht answers + Kommunikations-Checkliste persisted via wizard state. Textarea edits: Claude's discretion per D-08 context — recommend persisting in KommunikationData for consistency. |
