# Phase 06 Plan 01: Production Build verifizieren

## Objective

Sicherstellen dass `next build` mit `output: 'export'` sauber durchläuft — keine Errors, keine Build-Warnings. Turbopack-Root-Warning in next.config.ts beheben. `npm run build` muss ohne jegliche Warnungen oder Fehler beenden.

## Context

- @next.config.ts — aktuelle Build-Konfiguration (`output: 'export'`)
- @package.json — Build-Scripts
- @src/app/layout.tsx — Root-Layout

## Tasks

### Task 1: Turbopack-Root-Warning beheben
type: auto

Der Build gibt folgende Warnung aus:
```
Warning: Next.js inferred your workspace root, but it may not be correct.
We detected multiple lockfiles and selected the directory of C:\Users\PascalAmmeter\package-lock.json as the root directory.
To silence this warning, set `turbopack.root` in your Next.js config
```

Fix: `turbopack: { root: __dirname }` in `next.config.ts` setzen.

**Done criteria:** `npm run build` gibt keine Turbopack-Root-Warnung mehr aus.

### Task 2: Build-Output Verifikation
type: auto

Sicherstellen dass:
1. `npm run build` erfolgreich durchläuft (Exit-Code 0)
2. `out/` Verzeichnis mit statischen HTML/JS-Dateien existiert
3. Keine TypeScript-Fehler vorhanden

**Done criteria:** `out/index.html` existiert, Build-Output zeigt `✓ Generating static pages`.

## Verification

```bash
npm run build
ls out/
```

## Success Criteria

- `npm run build` läuft durch ohne Errors oder Warnings
- `out/` Verzeichnis mit `index.html` vorhanden
- Static export korrekt: Route `/` als `○ (Static)` markiert
