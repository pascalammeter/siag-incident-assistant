# 09-03 PLAN: Incident List UI & Integration

**Objective:** Build a fully functional incident list page at `/incidents` that displays all stored incidents with dynamic filtering, sorting, and action handlers. Users can view incident details, export, or delete directly from the list.

**Purpose:** Provide the central hub for incident management in v1.1. Users can see all their incidents, understand status at a glance, and quickly resume work on in-progress incidents or review completed ones.

**Output:** Complete `/incidents` route with reusable components, fully integrated with useIncident() hook, ready for styling refinement in Phase 10.

## Context

### UI Structure

```
/incidents (page.tsx)
  ├─ IncidentList (main container)
  │  ├─ FilterBar (type, severity filters + sort controls)
  │  ├─ LoadingState (skeleton while fetching)
  │  ├─ EmptyState (no incidents)
  │  └─ IncidentTable (list of incidents)
  │     └─ IncidentActions (View, Export, Delete buttons per row)
```

### Display Requirements

**Columns (Table):**
1. **Date** — createdAt, formatted as "2026-04-07 14:30" (user's timezone), clickable to View
2. **Type** — incident_type, badge with icon (Ransomware 🔴, Phishing 🎣, DDoS 🌊, Data Loss 📁, Other ❓)
3. **Severity** — severity badge (Critical 🔴, High 🟠, Medium 🟡, Low 🟢)
4. **Title** — first 50 chars of erkennungszeitpunkt or erkannt_durch or "Untitled", truncate with ellipsis
5. **Status** — inferred from wizard state: if incident_type is set but playbook incomplete → "In Progress", if all playbook steps checked → "Completed", if just created → "Draft"
6. **Actions** — buttons (View, Export, Delete)

**Filters:**
- **Type dropdown:** All Types, Ransomware, Phishing, DDoS, Data Loss, Other (default: All)
- **Severity dropdown:** All Severities, Critical, High, Medium, Low (default: All)

**Sorting:**
- Default: Date DESC (newest first)
- Also support: Type A→Z, Severity (Critical → Low)
- Sort control: dropdown or toggle buttons in FilterBar

**Loading States:**
- Skeleton loader for table (8 rows of placeholders) while fetching
- Spinning indicator + "Loading incidents..."

**Empty State:**
- "No incidents yet" message
- "Create one" link pointing to `/wizard` (entrance page, Step 1)

**Actions:**
- **View:** Router.push(`/wizard?incident=${id}`) — wizard loads incident via useIncident().getIncident(id) on mount
- **Export:** Placeholder for Phase 10 PDF export (for now: button disabled with tooltip "PDF export coming soon")
- **Delete:** Modal confirmation → useIncident().deleteIncident(id) → refresh list

### API Integration

- On mount: `useIncident().listIncidents()` → returns { data: Incident[], total: number }
- On filter change: re-fetch with filters parameter
- On action: call useIncident() method (create, get, update, delete)
- Error handling: show toast with error message (use sonner)
- Offline fallback: if API unavailable, show cached incidents from localStorage (useIncident hook handles this)

### Styling Requirements

Use Tailwind CSS with SIAG design system (Phase 7 colors):
- Navy (#003B5E) for headers and primary text
- Red (#CC0033) for critical severity and delete actions
- Orange (#D44E17) for high severity and warning actions
- Responsive: stack on mobile (<768px), flex on tablet, full table on desktop
- Card shadows for elevation
- Hover states on rows (gray background)
- Badge styles for type and severity

No animations in Phase 9 (defer to Phase 10).

Component interface:

```typescript
'use client'

export function IncidentList() {
  // State management
  const [filters, setFilters] = useState({ type: undefined, severity: undefined })
  const [sortBy, setSortBy] = useState<'date' | 'type' | 'severity'>('date')
  const [sortOrder, setSortOrder] = useState<'asc' | 'desc'>('desc')
  
  // API integration
  const { incident: _, isLoading, error, listIncidents } = useIncident()
  const [incidents, setIncidents] = useState<Incident[]>([])
  
  // Lifecycle
  useEffect(() => {
    fetchIncidents()
  }, [filters, sortBy, sortOrder])
  
  async function fetchIncidents() {
    try {
      const result = await listIncidents(filters)
      setIncidents(result)
    } catch (err) {
      toast.error('Failed to load incidents')
    }
  }
  
  // Handlers
  const handleFiltersChange = (newFilters) => setFilters(newFilters)
  const handleSortChange = (sort, order) => { setSortBy(sort); setSortOrder(order) }
  const handleViewClick = (id: string) => router.push(`/wizard?incident=${id}`)
  const handleDeleteClick = async (id: string) => {
    await useIncident().deleteIncident(id)
    setIncidents(incidents.filter(i => i.id !== id))
    toast.success('Incident deleted')
  }
  
  return (
    <div>
      <FilterBar filters={filters} sortBy={sortBy} sortOrder={sortOrder} ... />
      {isLoading && <LoadingState />}
      {error && <ErrorBanner message={error} />}
      {incidents.length === 0 && !isLoading && <EmptyState />}
      {incidents.length > 0 && <IncidentTable incidents={incidents} ... />}
    </div>
  )
}
```

**Implementation details:**

1. **State management:**
   - Use useState for filters, sortBy, sortOrder
   - Use useState for incidents list
   - Track isLoading and error from useIncident() hook

2. **useEffect:**
   - Trigger on filters/sort changes
   - Call `useIncident().listIncidents(filters)` with filter params
   - Store results in local state
   - Handle loading/error states

3. **Event handlers:**
   - `handleFiltersChange`: update filter state (triggers refetch via useEffect)
   - `handleSortChange`: update sort state
   - `handleViewClick`: use Next.js useRouter to push `/wizard?incident={id}`
   - `handleDeleteClick`: call useIncident().deleteIncident(), remove from list, show toast
   - `handleExportClick`: no-op for Phase 9 (Phase 10 adds PDF generation)

4. **Render logic:**
   - If isLoading: show LoadingState
   - If error: show error banner (toast or inline alert)
   - If incidents.length === 0 and not loading: show EmptyState
   - If incidents.length > 0: show FilterBar + IncidentTable

5. **Type safety:**
   - Import Incident, ListIncidentsFilters from incident-types.ts
   - Use useIncident() hook from src/hooks/useIncident.ts
   - Use useRouter from next/navigation

6. **Error handling:**

   - useIncident() hook already handles API errors and localStorage fallback
   - Show isOffline indicator if offline (optional, nice-to-have)
