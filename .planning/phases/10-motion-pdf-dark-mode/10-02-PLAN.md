# 10-02 PLAN: Dark Mode with next-themes + Tailwind v4

**Objective:** Implement comprehensive dark mode support using next-themes (v0.4.6) and Tailwind v4's CSS variable system. Users can toggle between light and dark modes via Header button; preference is persisted to localStorage and restored on page reload. All SIAG colors are adjusted for readability in dark mode.

**Purpose:** Provide users with a comfortable viewing experience in low-light conditions, reduce eye strain, and maintain brand consistency across light and dark color schemes.

**Output:** Fully functional dark mode with no hydration mismatches, localStorage persistence, and system preference detection.

Create a client component that provides a button to toggle between light and dark modes:

```typescript
'use client';

import { useTheme } from 'next-themes';
import { useEffect, useState } from 'react';

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();
  const [mounted, setMounted] = useState(false);

  // Hydration fix: only render after client-side mount
  useEffect(() => {
    setMounted(true);
  }, []);

  if (!mounted) {
    return <div className="w-10 h-10" />; // Placeholder to prevent layout shift
  }

  return (
    <button
      onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}
      className="relative w-10 h-10 rounded-lg bg-slate-100 dark:bg-slate-800 p-2 hover:bg-slate-200 dark:hover:bg-slate-700 transition-colors"
      aria-label="Toggle dark mode"
    >
      {theme === 'dark' ? (
        <svg className="w-full h-full text-slate-400" fill="currentColor" viewBox="0 0 24 24">
          {/* Sun icon */}
          <circle cx="12" cy="12" r="5" />
          <path d="M12 1v6m0 6v6M4.22 4.22l4.24 4.24m5.08 5.08l4.24 4.24M1 12h6m6 0h6M4.22 19.78l4.24-4.24m5.08-5.08l4.24-4.24" 
                strokeWidth="2" stroke="currentColor" fill="none" />
        </svg>
      ) : (
        <svg className="w-full h-full text-slate-600" fill="currentColor" viewBox="0 0 24 24">
          {/* Moon icon */}
          <path d="M21 12.79A9 9 0 1 1 11.21 3 7 7 0 0 0 21 12.79z" />
        </svg>
      )}
    </button>
  );
}
```

Key points:
- Use `useTheme()` hook from next-themes
- Check `mounted` state before rendering (hydration safety)
- Render placeholder div while not mounted (prevents layout shift)
- Toggle between 'light' and 'dark' themes
- Include sun icon for light mode, moon icon for dark mode
- Use semantic SVG icons (or use lucide-react if available)
- Button should be styled to match Header design
- `aria-label` for accessibility

Alternative (if you prefer text):
```typescript
<button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
  {theme === 'dark' ? '☀️ Light' : '🌙 Dark'}
</button>
```

```css
/* Tailwind v4 dark mode variant definition */
@custom-variant dark (&:where(.dark, .dark *));

/* Dark mode color overrides */
@media (prefers-color-scheme: dark) {
  @theme {
    --color-siag-red: #FF3355;          /* #CC0033 → #FF3355 (lighter for contrast) */
    --color-siag-navy: #1E3A8A;         /* #003B5E → #1E3A8A (lighter) */
    --color-siag-orange: #FB923C;       /* #D44E17 → #FB923C (lighter) */
    --color-background: #0f172a;        /* Slate 950 */
    --color-text-primary: #f1f5f9;      /* Slate 100 */
    --color-text-secondary: #cbd5e1;    /* Slate 300 */
    --color-border: #334155;            /* Slate 700 */
    --color-card-bg: #1e293b;           /* Slate 800 */
  }
}

/* Dark mode utility classes (if not using Tailwind dark: prefix) */
.dark {
  @apply bg-slate-950 text-slate-50;
}

/* Button dark mode */
.dark .btn-primary {
  @apply bg-siag-red hover:bg-red-600 dark:bg-siag-red-dark;
}

/* Card dark mode */
.dark .card {
  @apply bg-slate-900 border-slate-700;
}

/* Link dark mode */
.dark a {
  @apply text-siag-red-dark hover:text-siag-red-light;
}

/* Print styles (apply to both light and dark) */
@media print {
  body {
    background: white;
    color: black;
  }
  .dark {
    background: white;
    color: black;
  }
}
```

- Dark colors should be lighter/brighter than light mode (e.g., #CC0033 → #FF3355)

- Add overrides for buttons, cards, links, and other interactive elements
- Print styles should always use dark text on white background
- Verify with contrast checker: https://webaim.org/resources/contrastchecker/

```typescript
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { RootLayout } from '@/app/layout';
import { ThemeToggle } from '@/components/atoms/ThemeToggle';

describe('Dark Mode', () => {
  beforeEach(() => {
    // Clear localStorage before each test
    localStorage.clear();
  });

  it('ThemeToggle component renders without errors', () => {
    render(<ThemeToggle />);
    const button = screen.getByRole('button');
    expect(button).toBeInTheDocument();
  });

  it('toggles between light and dark themes on click', async () => {
    const user = userEvent.setup();
    render(<ThemeToggle />);
    
    const button = screen.getByRole('button', { name: /toggle dark mode/i });
    await user.click(button);
    
    // After click, theme should change
    expect(screen.getByRole('button')).toBeInTheDocument();
  });

  it('persists theme preference to localStorage', async () => {
    const user = userEvent.setup();
    render(<ThemeToggle />);
    
    const button = screen.getByRole('button');
    await user.click(button);
    
    // Theme preference should be stored
    expect(localStorage.getItem('theme')).toBeDefined();
  });

  it('restores theme from localStorage on mount', () => {
    localStorage.setItem('theme', 'dark');
    
    render(<ThemeToggle />);
    
    // Component should load with dark theme preference
    expect(screen.getByRole('button')).toBeInTheDocument();
  });

  it('respects system dark mode preference when no localStorage', () => {
    // Mock matchMedia for dark mode preference
    Object.defineProperty(window, 'matchMedia', {
      writable: true,
      value: jest.fn().mockImplementation(query => ({
        matches: query === '(prefers-color-scheme: dark)',
        media: query,
      })),
    });
    
    render(<ThemeToggle />);
    expect(screen.getByRole('button')).toBeInTheDocument();
  });

  it('theme toggle has sufficient color contrast', () => {
    const { container } = render(<ThemeToggle />);
    const button = container.querySelector('button');
    
    // Verify button has visible styling
    expect(button).toHaveClass('dark:bg-slate-800');
  });

  it('dark mode styles are applied to body', () => {
    // This test verifies that dark: prefix works in Tailwind
    const { container } = render(
      <div className="bg-white dark:bg-slate-950 text-slate-900 dark:text-slate-50">
        Dark mode text
      </div>
    );
    
    expect(container.querySelector('div')).toBeInTheDocument();
  });

  it('print styles apply to dark mode elements', () => {
    // Verify @media print styles are present
    const styleSheets = Array.from(document.styleSheets);
    const hasPrintStyles = styleSheets.some(sheet => {
      try {
        return Array.from(sheet.cssRules).some(rule => 
          rule.media?.mediaText.includes('print')
        );
      } catch (e) {
        return false;
      }
    });
    
    // Print styles should exist
    expect(hasPrintStyles || styleSheets.length > 0).toBe(true);
  });
});
```
