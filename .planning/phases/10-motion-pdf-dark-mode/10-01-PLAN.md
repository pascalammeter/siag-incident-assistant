# 10-01 PLAN: Motion + Button States + Loading Spinner

**Objective:** Implement smooth, accessible animations throughout the SIAG Incident Assistant using the Motion library (v12.38.0). All interactive buttons will have hover (150ms ease-out) and press (100ms scale 0.98) states. Loading spinners will animate during API requests. All animations respect the user's prefers-reduced-motion accessibility setting.

**Purpose:** Provide visual feedback for user interactions, improve perceived performance during loading states, and maintain accessibility for users with motion sensitivity.

**Output:** Fully animated component suite (Button, Card, Spinner) integrated into root layout, tested for both animation presence and accessibility compliance.

## Context

### Foundation Already In Place

- **globals.css:** Already includes `@media (prefers-reduced-motion: reduce)` block (lines 168–190)
- **Tailwind v4:** Already installed with transition utilities (`duration-150`, `duration-100`, `ease-out`)
- **React 19 + Next.js 15:** Supports client components with `'use client'` directive
- **Design tokens:** SIAG colors already defined as CSS custom properties

### Motion Library (v12.38.0)

Motion is the rebranded Framer Motion. Key features:
- Hybrid animation engine (Web Animations API + JS fallback)
- Built-in `reducedMotion` support (automatic accessibility compliance)
- React 19 compatible
- Tree-shakeable bundle (only import what you use)

### Accessibility Non-Negotiables

1. `MotionConfig reducedMotion="user"` at app root automatically disables all Motion animations when `prefers-reduced-motion: reduce` is set
2. CSS-based spinner already respects prefers-reduced-motion via globals.css media query
3. All interactions must have visual feedback even without animation (e.g., `:active` pseudo-class)
4. Test with macOS Accessibility > Display > Reduce Motion enabled

### Animation Specifications (from ROADMAP)

- **Button Hover:** 150ms ease-out transition, slight scale increase (1.05) + shadow increase
- **Button Press/Tap:** 100ms scale transition, scale down to 0.98
- **Card Hover:** Shadow increase from 0 to md (or lg on larger cards)
- **Loading Spinner:** 12-frame rotation, 1s total duration, continuous loop

Implementation pattern:
```typescript
'use client';

import { motion } from 'motion/react';
import { ANIMATION_VARIANTS, ANIMATION_DURATIONS, ANIMATION_EASING } from '@/lib/motion-config';

export interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'danger';
  size?: 'sm' | 'md' | 'lg';
  isLoading?: boolean;
}

export function Button({
  variant = 'primary',
  size = 'md',
  isLoading = false,
  disabled,
  children,
  ...props
}: ButtonProps) {
  return (
    <motion.button
      initial={{ scale: 1 }}
      whileHover={{ scale: 1.05, boxShadow: '0 10px 25px rgba(0,0,0,0.1)' }}
      whileTap={{ scale: 0.98 }}
      transition={{
        hover: { duration: ANIMATION_DURATIONS.hover, ease: 'easeOut' },
        tap: { duration: ANIMATION_DURATIONS.tap, ease: 'easeOut' },
      }}
      disabled={isLoading || disabled}
      className={classNames(
        'relative font-medium transition-all duration-150',
        'active:scale-98 active:shadow-sm',  // Fallback for reduced-motion
        variantClasses[variant],
        sizeClasses[size]
      )}
      {...props}
    >
      {isLoading && <LoadingSpinner size="sm" className="mr-2" />}
      {children}
    </motion.button>
  );
}
```

Key points:
- Use `motion.button` instead of `<button>`
- `whileHover` and `whileTap` define animation states
- `transition` object specifies duration and easing
- Include `:active` CSS class for fallback (when animations disabled)
- Keep all existing variant logic (primary/secondary/danger colors)
- Support `isLoading` prop to show spinner inside button

Implementation:
```typescript
'use client';

import { motion } from 'motion/react';

export interface LoadingSpinnerProps {
  size?: 'sm' | 'md' | 'lg';
  className?: string;
}

export function LoadingSpinner({ size = 'md', className = '' }: LoadingSpinnerProps) {
  const sizeClasses = {
    sm: 'w-4 h-4',
    md: 'w-6 h-6',
    lg: 'w-8 h-8',
  };

  return (
    <motion.div
      animate={{ rotate: 360 }}
      transition={{ duration: 1, repeat: Infinity, ease: 'linear' }}
      className={`inline-block ${sizeClasses[size]} ${className}`}
    >
      <div className="w-full h-full border-2 border-current border-t-transparent rounded-full" />
    </motion.div>
  );
}
```

Key points:
- Use `animate={{ rotate: 360 }}` for continuous 360° rotation
- `repeat: Infinity` for endless loop
- `ease: 'linear'` for smooth, uninterrupted rotation (not easeOut)
- CSS border creates a circle with transparent top (standard spinner design)
- `border-current` uses the current text color (respects dark mode)
- Support sm/md/lg sizes
- No dependencies beyond motion/react

Accessibility note:
- `prefers-reduced-motion` is automatically handled by `MotionConfig` in layout
- When reduced-motion is enabled, the spinner will not animate, but will still be visible (static circle)
