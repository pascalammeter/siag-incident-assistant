# 10-03 PLAN: PDF Export with Puppeteer + Title Page

**Objective:** Implement server-side PDF generation for incident export using Puppeteer (v24.40.0) with HTML templates. PDFs include a professional title page with SIAG logo and incident metadata, followed by multi-page incident details. All styling is optimized for print (no bright backgrounds, readable fonts, professional appearance). Client-side export button triggers API call and displays LoadingSpinner during generation.

**Purpose:** Enable users to export completed incidents as professional PDF reports suitable for compliance documentation, stakeholder communication, and offline archiving.

**Output:** Fully functional PDF export endpoint tested on Vercel Functions with <30 second cold-start handling.

### Serverless Constraints

Vercel Functions have:
- 10-second default timeout (60-second max with Pro)
- ~3GB max memory per invocation
- Limited file system (/tmp is writable, /var is ephemeral)
- Cold start: ~5-15 seconds to spin up Node.js runtime

Puppeteer cold start:
- Chromium binary load: ~10-15 seconds
- HTML rendering: ~2-5 seconds
- PDF generation: ~1-2 seconds
- Total: 15-25 seconds under ideal conditions

**Strategy:** Use `@sparticuz/chromium-min` (50MB minimal Chromium) instead of full Chromium (200MB+), implement singleton browser instance to reuse across requests, use `waitUntil: 'networkidle0'` to avoid waiting for all network requests.

From ROADMAP:
- No bright backgrounds (white background, dark text)
- Minimum 12pt font size for body text
- Professional layout with proper spacing
- Page breaks between sections
- Headers/footers with page numbers (optional but recommended)

```typescript
import puppeteer, { Browser } from 'puppeteer';
import chromium from '@sparticuz/chromium-min';

let browserInstance: Browser | null = null;

/**
 * Get or create a Puppeteer browser instance.
 * Reuses existing instance across invocations (serverless optimization).
 */
export async function getBrowserInstance(): Promise<Browser> {
  if (browserInstance) {
    return browserInstance;
  }

  const isProduction = process.env.NODE_ENV === 'production';

  browserInstance = await puppeteer.launch({
    args: isProduction
      ? chromium.args
      : [
          '--no-sandbox',
          '--disable-setuid-sandbox',
          '--disable-dev-shm-usage',
        ],
    executablePath: isProduction ? await chromium.executablePath : undefined,
    headless: true,
  });

  return browserInstance;
}

/**
 * Close the browser instance (called on shutdown or error).
 */
export async function closeBrowser(): Promise<void> {
  if (browserInstance) {
    await browserInstance.close();
    browserInstance = null;
  }
}

/**
 * Reset the singleton (for testing).
 */
export function resetBrowser(): void {
  browserInstance = null;
}
```

Key points:
- Singleton pattern reuses browser instance across multiple requests
- On first invocation, launches browser (~10-15 seconds cold start)
- On subsequent invocations, reuses existing instance (fast, <100ms)
- `@sparticuz/chromium-min` provides 50MB minimal Chromium binary
- In production, uses chromium-min; in dev, uses system Chromium
- No sandbox mode for Vercel compatibility (security handled by Vercel runtime)
- `headless: true` required (no display server available)

export function generateCompletePDF(incident: Incident): string {

  const details = generateIncidentDetailsHTML(incident);

  // Extract body content from both templates
  const titleMatch = titlePage.match(/<body[^>]*>([\s\S]*?)<\/body>/);
  const detailsMatch = details.match(/<body[^>]*>([\s\S]*?)<\/body>/);

  const titleContent = titleMatch ? titleMatch[1] : '';
  const detailsContent = detailsMatch ? detailsMatch[1] : '';

  return `
    <!DOCTYPE html>
    <html>
      <head>
        <meta charset="UTF-8">
        <style>
          * { margin: 0; padding: 0; box-sizing: border-box; }
          body { font-family: 'Segoe UI', sans-serif; color: #333; font-size: 12pt; }

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { getBrowserInstance } from '@/lib/puppeteer-singleton';
import { generateCompletePDF } from '@/lib/pdf-templates';
import { IncidentAPI } from '@/hooks/useIncidentAPI';

export async function GET(
  req: NextRequest,
  { params }: { params: { id: string } }
) {
  try {
    const { id } = params;

    // 1. Fetch incident from API (reuse existing endpoint)
    let incident;
    try {
      incident = await IncidentAPI.getIncident(id);
    } catch (error) {
      return NextResponse.json(
        { error: 'Incident not found' },
        { status: 404 }
      );
    }

    // 2. Generate HTML from template
    const htmlContent = generateCompletePDF(incident);

    // 3. Launch Puppeteer and render PDF
    const browser = await getBrowserInstance();
    const page = await browser.newPage();

    // Set page size and margins
    await page.setContent(htmlContent, { waitUntil: 'networkidle0' });

    // Generate PDF
    const pdfBuffer = await page.pdf({
      format: 'A4',
      margin: { top: 20, right: 20, bottom: 20, left: 20 },
      printBackground: false, // Don't render backgrounds (lighter PDF, faster)
    });

    // Close page (keep browser open for next request)
    await page.close();

    // 4. Return PDF as response
    const filename = `incident-${id}-${new Date().toISOString().split('T')[0]}.pdf`;

    return new NextResponse(pdfBuffer, {
      status: 200,
      headers: {
        'Content-Type': 'application/pdf',
        'Content-Disposition': `attachment; filename="${filename}"`,
        'Cache-Control': 'no-cache, no-store, must-revalidate',
      },
    });
  } catch (error) {
    console.error('PDF export error:', error);
    return NextResponse.json(
      { error: 'Failed to generate PDF. Please try again.' },
      { status: 500 }
    );
  }
}
```

Key points:
- Route path: `/api/incidents/[id]/export-pdf` (dynamic incident ID)
- HTTP method: GET (simplifies CORS and caching)
- Fetch incident from database via `IncidentAPI.getIncident(id)`
- Generate HTML using templates
- Use singleton browser instance (for serverless optimization)
- `waitUntil: 'networkidle0'` avoids waiting for background requests
- `printBackground: false` reduces PDF size and rendering time
- Return PDF with proper headers (`Content-Type: application/pdf`, `Content-Disposition: attachment`)
- Filename includes incident ID and date for easy identification
- Error handling: return 404 if incident not found, 500 if PDF generation fails
- Keep page open — don't close browser (singleton pattern reuses across requests)

'use client';

import { useState } from 'react';
import { Button } from '@/components/atoms/Button';
import { LoadingSpinner } from '@/components/atoms/LoadingSpinner';
import { Incident } from '@/lib/incident-types';

export interface IncidentActionsProps {
  incident: Incident;
  onDelete?: (id: string) => Promise<void>;
}

export function IncidentActions({ incident, onDelete }: IncidentActionsProps) {
  const [isExporting, setIsExporting] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const handleExportPDF = async () => {
    setIsExporting(true);
    setError(null);

    try {
      // Fetch PDF from API
      const response = await fetch(`/api/incidents/${incident.id}/export-pdf`);

      if (!response.ok) {
        const errorData = await response.json();
        throw new Error(errorData.error || 'Failed to export PDF');
      }

      // Create blob and download
      const blob = await response.blob();
      const url = window.URL.createObjectURL(blob);
      const link = document.createElement('a');
      link.href = url;
      link.download = `incident-${incident.id}-${new Date().toISOString().split('T')[0]}.pdf`;
      document.body.appendChild(link);
      link.click();
      document.body.removeChild(link);
      window.URL.revokeObjectURL(url);
    } catch (err) {
      const message = err instanceof Error ? err.message : 'Unknown error';
      setError(message);
      console.error('PDF export failed:', err);
    } finally {
      setIsExporting(false);
    }
  };

  return (
    <div className="flex items-center gap-3">
      {error && (
        <div className="text-sm text-red-600 dark:text-red-400">
          {error}
        </div>
      )}

      {isExporting && <LoadingSpinner size="sm" />}

      <Button
        onClick={handleExportPDF}
        disabled={isExporting}
        variant="primary"
      >
        {isExporting ? 'Exporting...' : 'Download PDF'}
      </Button>

      {onDelete && (
        <Button onClick={() => onDelete(incident.id)} variant="danger">
          Delete
        </Button>
      )}
    </div>
  );
}
