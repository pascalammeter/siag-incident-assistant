**Replace the placeholder Incident model with the complete version:**

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")
}

model Incident {
  id            String    @id @default(uuid())
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  // ========== Recognition (Erkennung) ==========
  erkennungszeitpunkt DateTime?
  erkannt_durch       String?  @db.VarChar(255)
  betroffene_systeme  String[] @db.Text
  erste_erkenntnisse  String?  @db.Text

  // ========== Classification (Klassifizierung) ==========
  incident_type String?  @db.VarChar(50) // "ransomware" | "phishing" | "ddos" | "data_loss" | "other"
  q1            Int?     // Classification question 1 (0-1)
  q2            Int?     // Classification question 2 (0-1)
  q3            Int?     // Classification question 3 (0-1)
  severity      String?  @db.VarChar(20) // "critical" | "high" | "medium" | "low"

  // ========== Playbook & Progress (JSONB) ==========
  // Structure: { checkedSteps: [{ stepId: string, checked: boolean, timestamp: ISO8601 }], status: "in_progress"|"completed" }
  playbook      Json?    @db.JsonB

  // ========== Regulatory (Regulatorische Meldungen) ==========
  // Structure: { isg_24h: ISO8601 datetime, dsg: boolean, finma_24h: ISO8601, finma_72h: ISO8601 }
  regulatorische_meldungen Json? @db.JsonB

  // ========== Metadata & Audit Trail ==========
  // Structure: { tags: [string], notes: string, custom_fields: Record<string, any> }
  metadata      Json?    @db.JsonB

  // ========== Indexes for Query Performance ==========
  @@index([incident_type])
  @@index([severity])
  @@index([createdAt(sort: Desc)])
  @@index([erkennungszeitpunkt(sort: Desc)])
}
```

**Rationale for field design (from 07-RESEARCH.md):**
- **Recognition fields:** Store when and how the incident was discovered, which systems affected, initial observations
- **Classification fields:** incident_type (enum), q1/q2/q3 (classification questionnaire answers), severity level
- **Playbook JSONB:** Stores checked steps with timestamps for each playbook (no separate table; flexible for 4 incident types)
- **Regulatory JSONB:** ISG 24-hour notification deadline, DSG flag, FINMA 24h/72h deadlines (calculated from erkennungszeitpunkt)
- **Metadata JSONB:** Tags, notes, custom fields for future extensibility without schema migrations
- **Indexes on relational columns:** incident_type, severity, and createdAt for efficient filtering and sorting in list view

**Update src/lib/prisma.ts with enhanced logging:**
```typescript
import { PrismaClient } from '@prisma/client';

const globalForPrisma = global as unknown as { prisma: PrismaClient };

export const prisma =
  globalForPrisma.prisma ||
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' 
      ? [
          { level: 'query', emit: 'event' },
          { level: 'error', emit: 'event' },
          { level: 'warn', emit: 'event' },
        ]
      : [
          { level: 'error', emit: 'event' },
        ],
  });

// Development: Log all queries and timing
if (process.env.NODE_ENV === 'development') {
  prisma.$on('query', (e: any) => {
    console.log(`[${new Date().toISOString()}] Query took ${e.duration}ms`);
    console.log(`  ${e.query}`);
  });
}

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma;
}

export default prisma;
```

**Why this matters:**
- Query logging helps debug performance issues in Phase 08
- Timing shows which queries are slow (candidates for indexing)
- Development mode shows all queries; production only logs errors

**Disable query logging if it's noisy:**
Set `PRISMA_LOG_QUERIES=false` in .env.local, or comment out the `$on('query')` block above.

## Threat Model

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | Database connection strings in .env.local | mitigate | Use .env.local (gitignored); .env.example provided without secrets; Neon credentials never committed |
| T-07-02 | Access Control | Direct database connection exposed | mitigate | DIRECT_URL shared only with Prisma CLI; APPLICATION uses pooled connection via DATABASE_URL; environment variables isolate credentials |
| T-07-03 | Data Integrity | SQL injection via Prisma ORM | mitigate | Parameterized queries only; Prisma ORM prevents direct string concatenation; Zod validation prevents malformed input reaching DB |
| T-07-04 | Denial of Service | Connection pool exhaustion | mitigate | Neon PgBouncer limits connections per role; application uses singleton PrismaClient to minimize open connections; rate limiting in middleware |
