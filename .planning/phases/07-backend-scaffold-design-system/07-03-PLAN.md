- Implements SIAG visual design system (colors, typography, motion)

```typescript
import type { Metadata } from 'next';
import { Source_Sans_Pro } from 'next/font/google';
import './globals.css';

// Load Source Sans Pro from Google Fonts
const sourceSansPro = Source_Sans_Pro({
  subsets: ['latin', 'latin-ext'],
  weight: ['400', '500', '600', '700'],
  variable: '--font-sans',
  display: 'swap', // Show fallback while loading
  fallback: ['system-ui', 'sans-serif'],
});

export const metadata: Metadata = {
  title: 'SIAG Incident Assistant',
  description: 'Swiss incident management and regulatory response platform',
  viewport: 'width=device-width, initial-scale=1',
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html
      lang="en"
      suppressHydrationWarning // Needed for dark mode toggle (Phase 10)
      className={sourceSansPro.variable}
    >
      <body className="bg-white text-slate-900 dark:bg-slate-950 dark:text-slate-50 transition-colors">
        {children}
      </body>
    </html>
  );
}
```

**Optional: Add Stone Sans Pro via @font-face**

If SIAG provides the Stone Sans Pro font file (e.g., `stone-sans-pro.woff2`):

1. Place font files in `public/fonts/`
2. In globals.css (Task 3), add:

```css
@font-face {
  font-family: 'Stone Sans Pro';
  src: url('/fonts/stone-sans-pro-regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}

@font-face {
  font-family: 'Stone Sans Pro';
  src: url('/fonts/stone-sans-pro-bold.woff2') format('woff2');
  font-weight: 700;
  font-display: swap;
}
```

**Fonts loaded:**
- **Source Sans Pro** (400, 500, 600, 700) — body text, labels, UI
- **Stone Sans Pro** (optional, 400/700) — display headings (H1, H2)

**CSS variables injected:**
- `--font-sans`: Source Sans Pro (injected by next/font/google)

**Performance note:**
- `font-display: swap` ensures text is visible immediately using system font, swapping to loaded font when ready
- Google Fonts fonts are served from Google's CDN (fast, cached globally)
- Stone Sans Pro (if self-hosted) should be in `public/` for optimal loading

**Font hierarchy (will be applied in globals.css):**
- H1/H2: Stone Sans Pro 700 (display)
- H3/body: Source Sans Pro 400-700 (functional)
- Labels/buttons: Source Sans Pro 500-600
