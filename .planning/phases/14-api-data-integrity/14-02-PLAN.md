# Phase 14 Plan 02: Fix IncidentService Field Persistence

## Objective

Update IncidentService to persist all incident fields passed from the wizard, eliminating silent data loss during Prisma writes.

## Execution Context

**Current State:**
- IncidentService.createIncident() (src/api/services/incident.service.ts, lines 6-18) only persists 5 fields
- Wizard input includes 14 fields total but service drops 9 of them before Prisma write
- updateIncident() (lines 31-55) has similar selective persistence

**Problem:**
- Wizard provides: erkennungszeitpunkt, erkannt_durch, erste_erkenntnisse, betroffene_systeme, q1, q2, q3, playbook, regulatorische_meldungen, metadata, etc.
- Service ignores: erkennungszeitpunkt, erkannt_durch, erste_erkenntnisse, betroffene_systeme, q1, q2, q3
- Result: Fields in database are null/empty when they should contain wizard data
- Data retrieved from DB lacks incident context needed for playbook routing and reporting
